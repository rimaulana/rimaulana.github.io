Mengurutkan Interface Pada Virtualisasi Router Mikrotik Di Qemu

2017-08-30 463 words 3 min read

Table of Contents

Pastinya kita semua pernah mengalami pada saat sedang konsennya konfig mikrotik dan telah selesai dan yakin 100% konfig kita benar, lalu saat kita coba ping … . . gagal. destination host unreachable. Mungkin salah satu penyebabnya adalah interface yang kita gunakan adalah bukan interface yang benar. Saya sering mengalami hal ini saat menggunakan GNS3 dengan, bisa dibilang hampir semua virtualisasi (VirtualBox dan Qemu) kecuali VMware. Cara yang mudah untuk melakukan pemeriksaan, bisa kita lihat dari MAC address interface melalui command

/interface ethernet print

Dimana salah satu contoh outputnya adalah sebagai berikut:

Flags: X - disabled, R - running, S - slave 
 #    NAME        MTU      MAC-ADDRESS         ARP            
 0 R  ether1      1500     00:BB:0E:E7:AE:01   enabled        
 1 R  ether2      1500     00:BB:0E:E7:AE:02   enabled        
 2 R  ether3      1500     00:BB:0E:E7:AE:03   enabled        
 3 R  ether4      1500     00:BB:0E:E7:AE:04   enabled        
 4 R  ether5      1500     00:BB:0E:E7:AE:05   enabled        
 5 R  ether6      1500     00:BB:0E:E7:AE:06   enabled        
 6 R  ether7      1500     00:BB:0E:E7:AE:07   enabled        
 7 R  ether8      1500     00:BB:0E:E7:AE:00   enabled

Contoh diatas saya dapatkan dari VM mikrotik yang berjalan diatas Qemu. Pada umumnya, untuk virtualisasi menggunakan Qemu nomor MAC address selalu urut dari 00 hinggi NN, sehingga dengan assumsi tersebut, hasil output diatas menyatakan bahwa urutan interface pada router tersebut tidak urut dan perlu dibenahi. Seharusnya urutannya adalah:

Interface MAC Address
ether1 00:BB:0E:E7:AE:00
ether2 00:BB:0E:E7:AE:01
ether3 00:BB:0E:E7:AE:02
ether4 00:BB:0E:E7:AE:03
ether5 00:BB:0E:E7:AE:04
ether6 00:BB:0E:E7:AE:06

Untuk memudahkan pengurutan interface berasarkan MAC address kita bisa menggunakan command yang mana kita akan mengganti nama interface dengan nama yang lain kemudian kita ganti kembali nama interface sesuai dengan penamaan mikrotik, yakni etherX. Mengapa demikian? karena jika kita langsung mengganti nama interface menjadi etherX, bisa jadi nama tersebut sudah ada dan proses penggantian nama interface tersebut akan gagal. Command yang biasa saya gunakan adalah:

/interface ethernet 
set [find mac-address=AA:BB:CC:DD:EE:00] name=ethernet1
set [find mac-address=AA:BB:CC:DD:EE:01] name=ethernet2
set [find mac-address=AA:BB:CC:DD:EE:02] name=ethernet3
set [find mac-address=AA:BB:CC:DD:EE:03] name=ethernet4
set [find mac-address=AA:BB:CC:DD:EE:04] name=ethernet5
set [find mac-address=AA:BB:CC:DD:EE:05] name=ethernet6
set [find mac-address=AA:BB:CC:DD:EE:06] name=ethernet7
set [find mac-address=AA:BB:CC:DD:EE:07] name=ethernet8
set [find mac-address=AA:BB:CC:DD:EE:00] name=ether1
set [find mac-address=AA:BB:CC:DD:EE:01] name=ether2
set [find mac-address=AA:BB:CC:DD:EE:02] name=ether3
set [find mac-address=AA:BB:CC:DD:EE:03] name=ether4
set [find mac-address=AA:BB:CC:DD:EE:04] name=ether5
set [find mac-address=AA:BB:CC:DD:EE:05] name=ether6
set [find mac-address=AA:BB:CC:DD:EE:06] name=ether7
set [find mac-address=AA:BB:CC:DD:EE:07] name=ether8

Kita bisa dengan mudah mereplace (biasanya tekan ctrl+h) semua AA:BB:CC:DD:EE dengan prefix dari MAC address router, mengacu contoh diatas prefix MAC addressnya adalah 00:BB:0E:E7:AE. Setelah selesai maka kita akan mendapati MAC address kita akan urut sesuai penomoran di tabel

Flags: X - disabled, R - running, S - slave 
 #    NAME        MTU      MAC-ADDRESS         ARP            
 0 R  ether1      1500     00:BB:0E:E7:AE:00   enabled        
 1 R  ether2      1500     00:BB:0E:E7:AE:01   enabled        
 2 R  ether3      1500     00:BB:0E:E7:AE:02   enabled        
 3 R  ether4      1500     00:BB:0E:E7:AE:03   enabled        
 4 R  ether5      1500     00:BB:0E:E7:AE:04   enabled        
 5 R  ether6      1500     00:BB:0E:E7:AE:05   enabled        
 6 R  ether7      1500     00:BB:0E:E7:AE:06   enabled        
 7 R  ether8      1500     00:BB:0E:E7:AE:07   enabled

Selamat mencoba dan jangan sungkan untuk meninggalkan pesan apabila ada pertanyaan. Terima Kasih.
